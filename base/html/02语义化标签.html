# HTML 语义化标签

# 介绍

语义类标签是工作中常用到的标签，特点是视觉表现上都差不多，区别在于表示的不同语义

# 优点

  • 语义类标签对开发者更为友好，使用语义类标签增强了可读性，即便是在没有 CSS 的时候，开发者也能够清晰地看出网页的结构，也更为便于团队的开发和维护。
  • 除了对人类友好之外，语义类标签也十分适宜机器阅读。它的文字表现力丰富，更适合搜索引擎检索（SEO），也可以让搜索引擎爬虫更好地获取到更多有效信息，有效提升网页的搜索量，并且语义类还可以支持读屏软件，根据文章可以自动生成目录等等。

# 场景

  1. 自然语言表达能力的补充；
  2. 文章标题摘要；
  3. 适合机器阅读的整体结构。

# 内容

  1. header 出现在头部，表示导航或者介绍性的内容
  2. footer 出现在尾部，表示包含一些作者信息、相关链接、版权信息等
  3. main 表示主要内容，整个页面只出现一次
  4. article 表示文章主体部分，具有明确的独立性
  5. h1~h6 表示文章的各级标题
  6. aside 表示侧边栏导航性质的工具内容
  7. abbr 表示缩写
<abbr title="World Wide Web">WWW</abbr>
1
  1. hr 表示横向分割线，也表示话题的转变和故事走向的转变
  2. p 表示段落文本
  3. blockquote 表示段落级别的引述
  4. q 表示行内级别的引述
  5. cite 表示引述的作品名
  6. time 表示日期，让机器阅读更加方便

Retrieved <time datetime="2015-07-16">16 July 2015</time>.

# 缺点

  1. 自带标签样式，有时候并不需要，所以要先统一样式
  2. 页面元素语义不明确，导致不知道该用哪个标签
  3. 对语义标签理解不够深入的话，容易误用
