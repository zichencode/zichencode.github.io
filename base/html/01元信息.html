    # 元信息标签

    # 1.介绍

    是 HTML 用于描述文档自身的一类标签，它们通常出现在 head 标签中，一般都不会在页面被显示出来。

    • 主要是给浏览器、搜索引擎阅读的

    # 2.内容

    # 2.1 head 标签

    自身必须是html标签的第一个标签，否则浏览器会自己创建一个新的标签，且内容必须包含一个title

    # 2.2 title 标签

    表示文档的标题，完整的概括网页内容

    • 只包含文本，且一个head只能有一个title
    • 对SEO有重要意义，字符在55-60之间
    # 2.3 meta 标签

    是一组键值对，一种通用的元信息表示标签 在 head 中可以出现任意多个 meta 标签。一般的 meta 标签由 name 和 content 两个属性来定义。name 表示元信息的名，content 则用于表示元信息的值。

    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    1

    类别：

    • 如果设置了 charset 属性，meta 元素是一个字符集声明，告诉文档使用哪种字符编码。
    • 如果设置了 http-equiv 属性，meta 元素则是编译指令，提供的信息与类-似命名的HTTP头部相同。
    • 如果设置了 name 属性，meta 元素提供的是文档级别（document-level）的元数据，应用于整个页面。
    • 如果设置了 itemprop 属性，meta 元素提供用户定义的元数据。

    详细介绍

    # 2.3.1

    具有charset 属性 表示HTML文档自身的编码格式,建议放在 head 第一个，可以最大程度保证不乱吗

    <meta charset="UTF-8">
    1
    # 2.3.2

    具有http-equiv属性 表示执行一个命令，这样meta就不需要name属性 具有如下命令：

    • content-type：如果使用这个属性，其值必须为text-html;charset=utf8.
    • default-style：指定默认样式表.
    • refresh：刷新(1)如果content是一个数字，表示重载页面时间间隔(s).(2)包含一个正整数并且跟着一个字符串 ';url=' 和一个合法的 URL，则是重定向到指定链接的时间间隔(s)
    <meta http-equiv="refresh" content="3;url=http://www.google.com">
    1
    • set-cookie：模拟http 头，设置cookie值,由于存在xss攻击，所以在Chrome65以后的版本中已经不再支持了
    • x-ua-compatible：模拟 http 头 x-ua-compatible，声明 ua 兼容性,其值必须为"IE=edge"
    • content-security-policy:模拟 http 头 content-security-policy，声明内容安全策略.内容策略主要指定允许的服务器源和脚本端点，这有助于防止跨站点脚本攻击。 详细链接
    # 2.3.3

    具有 name 属性， 与 content 属性一起使用，以名-值对的方式给文档提供元数据，其中 name 作为元数据的名称，content 作为元数据的值。

    1. keywords: 页面关键字，对 seo 场景非常关键
    2. description: 页面描述，这个属性可能被用于搜索引擎或者其他场合
    3. author: 页面作者
    4. generator: 生成页面所使用的工具，主要用于可视化编辑器，如果是手写 HTML 的网页，不需要加这个 meta。
    5. application-name：如果页面是 Web application，用这个标签表示应用名称。
    6. theme-color: 页面风格颜色，实际并不会影响页面，但是浏览器可能据此调整页面之外的 UI（如窗口边框或者 tab 的颜色）。
    7. viewport :提供有关视口初始大小的提示，仅供移动设备使用.
      • width: 页面宽度，可以是具体数字也可以是 device-width , 表示和设备宽度相等
      • height: 页面高度，可以是具体数字也可以是 device-height , 表示和设备高度相等
      • initial-scale: 初始缩放比例.
      • minimum-scale: 最小缩放比例.
      • maximum-scale: 最大缩放比例.
      • user-scalable: 是否允许用户缩放. 一个标准的 viewport:
    
<meta name="viewport" content="width=device-width,initial-scale=1,minimum-scale=1,maximum-scale=1,user-scalable=no">

    # 2.4 base 标签

    作用是给页面上所有的 URL 相对地址提供一个基础。

    <base target="_blank" href="http://www.example.com/">  
表示：给所有相对路径的url添加基础路径和 target属性
