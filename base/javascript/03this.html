# this 详解

相信大家或多或少对 this 有一些困惑，相信看完下文，你就基本理解 this 的作用和使用

  1. this 在JavaScript 中有着什么样的作用
  2. this 在JavaScript 中指向问题

this 指向：

  1. 在函数被调用时候确定指向。完全取决于函数调用的地方，而不是声明的地方（箭头函数除外）。
  2. 函数被调用时会创建一个执行上下文（context），包含在哪里被调用、调用方式、传参信息等，this就是这个记录（上下文）的一个属性，在函数执行过程中被用到

this 场景：

  1. 普通函数直接调用
  2. 对象方法调用
  3. 构造函数被 new 调用
  4. bind、call、apply 调用
  5. 箭头函数中 this

1.1 普通函数直接调用(默认绑定) - 示例

  • 例1
var str = 'ly';
function fn() {
  var str = 'fn';
  console.log(this); // window
  console.log(this.str); // ly
}
fn()

2.1 对象方法调用（隐式调用） - 示例

  • 隐式调用，即函数是否在某个上下文中调用，如果是的话，this 绑定就是那个上下文对象。
var str = 'ly';
var obj = {
  str: 'cc',
  fn: function() {
    console.log(this.str)
  }
}
obj.fn() // cc 隐式绑定 作为对象属性执行，则this指向改对象

3.1 bind、call、apply（显式绑定）

  • 通过 bind、call、apply 修改函数的this指向，第一个参数可以显式 绑定this
function fn(name, age) {
  this.name = name;
  this.age = age;
}
function Fn(name, age) {
  fn.call(this, name, age); // call 调用 比 apply性能要好一些 1.参数更符合内部使用 2.不用调用 create
  // fn.apply(this, [name, age]); // apply 调用
}
new Fn('ly', 26)

4.1 new 绑定

函数作为构造函数使用new调用时，this绑定是新创建的构造函数实例

function Fn() {
  console.log(this) // Fn
}
var fn = new Fn()

new 构建过程

  1. 创建一个新的对象;
  2. 构造函数的prototype 指向新对象的 proto;
  3. 将新对象赋值给当前的this;
  4. 执行构造函数;
  5. 注：函数中如果没有返回其他对象，那么new 表达式中函数调用会自动返回这个新的对象。返回其他值忽略;
